Apple Design Review · v2rayNG

像系统设置一样冷静，
而不是像 VPN 广告一样用力

基于 WWDC 设计原则审查当前首页、节点列表、底部导航与诊断对话框。 方向不是更大的电源钮，也不是蓝绿渐变英雄区，而是 iOS 设置 / 系统 VPN 的语法： 大标题、inset 分组、系统开关、次要信息退后。状态机已经成熟，缺的是克制的呈现。

结构 7/10 · 首页信息分层清晰
气质 4/10 · 描边卡片偏工具面板
动效与打断 2/10 · 几乎无 spring / 手势
材质与深度 4/10 · 无 blur、无分层
文案与状态 8/10 · Connecting/Running 语义完整

审查结论 · 按原则

下列问题按优先级排列。每条对应 Apple 设计原则中的具体条款，并指向下方效果图中的改法。

P0 · 清晰与熟悉

连接被做成「工具面板」，而不是系统级状态页

当前首页是描边卡片堆叠 + 表单 Switch，读起来像设置页草稿。 优雅的方向不是更大的按钮，而是 iOS 设置 / 系统 VPN 的语法： 大标题状态、分组列表、系统开关、次要信息退后。 避免圆钮 ON/OFF、渐变英雄区、国旗选服条——那些是廉价 VPN 的识别符。

Familiarity · Simplicity · Craft
P0 · 反馈

Connecting / Running 几乎只改文字与描边色

bg_home_status_card_running 只是浅绿底 + 绿描边； 开关本身没有 scale、没有 spring settle、没有 haptics。 连接是高风险异步任务，却缺少「进行中」的连续反馈与可中断感。

Response · Behavior over animation · Multimodal
P0 · 简化

节点行永久露出 分享 / 编辑 / 删除 三个图标

item_recycler_main 把次要操作钉在主列表上，挤压名称与延迟， 视觉噪音高，误触面大。iOS 做法是：主行只保留选择与延迟； 次要操作进「…」或长按 Context Menu / 左滑。

Simplicity · Purpose · Agency
P1 · 材质

卡片全靠 1dp 描边，无层级与模糊

状态卡、指标卡、模式卡、节点卡都是 stroke + radius。底部导航不透明 elevation 8dp。 结果是「表格式」而不是「浮层式」——内容无法在 chrome 下滚动， 也没有 scroll edge 渐隐。

Materials & depth · Familiarity
P1 · 空间一致性

模式切换与节点跳转缺少源锚定

模式用 OutlinedButton ToggleGroup，视觉重量与「连接」主任务抢戏。 点「当前节点」跳转订阅 Tab，没有从卡片生长出的 sheet / 过渡， 空间关系断裂（If it disappears one way, it should reappear the same way）。

Spatial consistency · Hint in gesture direction
P1 · 手势

列表无 spring、无 rubber-band 语义、无上下文菜单

节点列表是标准 RecyclerView + 描边 item。拖拽排序存在（ItemTouchHelper）， 但缺少松开时的 velocity handoff；没有长按菜单，没有左滑归档式操作。

Interruptibility · Velocity handoff · Rubber-banding
P1 · 版式

字阶混用 Material3 与 AppCompat，tracking 未按字号调整

首页用 Material3 Headline/Body；节点行用 TextAppearance.AppCompat.Small + 硬编码 11–12sp。 大标题缺少负 tracking；小字未加正 tracking。层次靠 bold 堆叠而非 weight + size + leading 的组合。

Typography · Craft
P2 · 安全与代理

诊断对话框信息密度高，但缺少「下一步做什么」

HideVPN 自检卡片状态分层不错（pill / section）， 但结论与行动（重试、打开模块、复制日志）距离远。 Agency：给选择，不要只给原始数据。

Agency · Responsibility · Clarity
保留

状态机语义已经扎实

Stopped / Connecting / Running / Unreachable / Reconnecting 与 stickyRunning、超时、自动测通，说明后端状态模型成熟。 设计债主要在「如何呈现」而非「是否知道状态」。

Purpose · Feedback kinds
保留

四 Tab 信息架构合理

首页 / 订阅 / 路由 / 更多，符合「主任务 + 资源 + 策略 + 设置」。 不建议再加第五 Tab；建议把「测全部 / 管理订阅」沉到订阅页工具栏。

Wayfinding · Familiarity

效果图 1 · 首页（已停止）

左：当前 UI。右：iOS 设置语法——大标题状态、系统开关行、inset 分组、指标一行三列。没有电源钮、没有渐变英雄区、没有 ON/OFF 圆盘。

01 · Familiarity

系统设置的语法

大标题写状态，不是产品 slogan。主操作是系统开关，不是圆形电源。用户不会觉得在用「那种 VPN」。

02 · Hierarchy

状态在上，细节退后

「未连接」是页面的唯一大字。模式、节点、指标都是 inset 分组，不与主状态抢戏。

03 · Craft

没有戏剧性装饰

无渐变英雄区、无国旗条、无 ON/OFF 圆盘。绿色只出现在「已连接」的小圆点与开关上。

04 · Metrics

一行三列，安静并列

地区 / 延迟 / 24h 用分隔线并排，不是三张描边小卡。未连接时数值用 secondary 灰色。

效果图 2 · 首页（运行中）

运行态不靠整屏变色。大标题改为「已连接」，开关变绿，状态点亮，副标题显示时长与测通结果。Connecting 时标题写「连接中…」，开关仍可关掉以取消。

05 · Feedback

状态用字与小点，不靠整屏变色

大标题「已连接」、绿点、绿开关。没有蓝紫→绿的英雄区切换。Connecting 时标题变「连接中…」，开关仍可关。

06 · Duration

时长进副标题

00:42:18 与「外网正常」并排在 meta 行，不做成巨大计时器。测通是蓝色文字链，贴在因果点上。

07 · Quiet metrics

数字有，但不喧哗

延迟用 tabular nums 与绿色；上下行若需要可放节点副标题，不单独做 Download/Upload 三格广告牌。

效果图 3 · 订阅节点列表

压缩行内动作，把分享/编辑/删除收进「…」与长按菜单；选中用描边光环而非背景染色；延迟用 tabular nums。

08 · Simplicity

一行只做一件事：选中

点整行 = 选中并可选自动连接。分享/编辑/删除进 ··· 与长按 UIMenu，命中区更大、误删更少。

09 · Selection

选中用环，不用脏底色

2pt 橙色描边 + 勾选圆，比 color_home_node_selected_bg 半透明橙底更干净，暗色模式也稳定。

10 · Gesture

长按 = 上下文菜单

对齐 iOS 系统列表：长按弹出毛玻璃菜单，可打断、可滑出取消。左滑可保留「删除」快捷。

效果图 4 · HideVPN 诊断（Sheet）

从 Alert 式长滚动改为可拖拽 Bottom Sheet；顶部结论 + 行动按钮，细节折叠在分组里。

11 · Agency

结论在上，行动在下

用户先看到「部分通过」和含义，再决定复制报告或重测。原始 ifaces 字符串默认折叠。

12 · Sheet physics

可拖拽、可打断

damping 0.8 / response 0.3；下拉超界 rubber-band；松手按 velocity 投影决定关闭或回弹。
13 · Scrim

模态才压暗，非阻塞不压暗

诊断是模态任务 → 28% scrim + 背景轻微缩放。设置内的并行面板则只靠材质分离。

效果图 5 · 深色模式

纯黑底 + elevated #1c1c1e 分组。开关仍是系统绿，大标题与分组列表保持与浅色相同的冷静语法——不引入渐变英雄区。

落地建议（不改业务语义）

可按三期推进，状态机与服务层保持不动。

Phase A · 1 周

首页设置式重排 + 节点行瘦身

首页改为大标题状态 + 系统开关行 + inset 分组（模式/节点）； 去掉描边状态卡与表单式堆叠；节点行隐藏 share/edit/remove，改为 overflow； 指标一行三列补回延迟。不引入电源钮或渐变英雄区。

Phase B · 1–2 周

材质与分组

去掉描边卡片，改 surface + 轻 elevation；BottomNav 半透明； 模式改为 SegmentedButton（M3）或自定义 segmented； 诊断对话框改 Material BottomSheet。

Phase C · 持续

动效与触感

Dynamicanimation / Motion 弹簧；Connecting 可取消； 长按 Context Menu；haptic 成功/失败； 尊重系统「减少动态效果」。
